### Changed

Future passes can grow RunState into a real container for the ~540 module-scope globals (#11) and migrate more bodies into Spawner/Wave/Camera incrementally. Build green, 80/80 parity tests + 38 Vitest assertions still pass.

- **Player.js dead `window.TutorialMode` guard** ‚ÄĒ `Player.dash()` and `Player.useSpecial()` both gated their tutorial event hooks behind `if (window.TutorialMode) TutorialMode.onDash/onAbility()`. `window.TutorialMode` was never set anywhere (TutorialMode.js doesn't self-expose, game.js doesn't expose it either), so the guard always failed and tutorial dash/ability progress events never fired ‚ÄĒ meaning the dash and ability tutorial objectives were impossible to complete. Fix: `Player.js` now `import { TutorialMode } from './TutorialMode.js'` directly (matches the Phase 10 explicit-imports pattern from MeleeSwipe), and the `window.TutorialMode` guard is dropped.
- **Player.js dead `window._syncSoundBiomeMusic` guard** ‚ÄĒ `Player.levelUp()` calls `window._syncSoundBiomeMusic()` to refresh Sound hero music after a level-up. The function exists in `game.js` but was never exposed to `window`, so the `typeof window._syncSoundBiomeMusic === 'function'` guard always failed and the music never updated. Added `window._syncSoundBiomeMusic = _syncSoundBiomeMusic` in the window-exports block.
- **Gray enemies / wrong enemy subtypes + DLC write-back** ‚ÄĒ `var wave`, `var enemiesKilledInWave`, `var bossActive`, and `var isPlayerDying` in `game.js` are declared with `// Exposed for DLC` comments, but ESM module-level `var` does not auto-assign to `window` (unlike classic scripts). `Enemy.js` reads bare `wave` ‚Üí `window.wave` (never set) ‚Üí fallback 1 ‚Üí all subtypes resolve as BASIC ‚Üí all enemies render gray. Fixed with `Object.defineProperties` on `window` inside `game.js`, defining getter/setter pairs for all four vars: getters return the live module variable; setters update it in place. Both read and write directions now work with zero per-mutation sync boilerplate. No changes to DLC or TutorialMode files needed.
