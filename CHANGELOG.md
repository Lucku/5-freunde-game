# Changelog

All notable changes to this project will be documented in this file, starting with version 1.0.0 and recent changes involved just before that release.

## [Unreleased]

### Added
- **Build pipeline** (#2) ‚Äî Vite 8 introduced as the dev server + production bundler. `npm run dev` boots Vite at `http://localhost:5173` with full-page reload on file save and CSS HMR for `main.css` (no reload, live style swap). `npm run build` produces a fingerprinted bundle in `dist/`. `npm start` (electron-forge) loads `dist/game.html` when present, falls back to source `game.html` otherwise ‚Äî keeps the existing dev flow working unchanged. `vite-plugin-electron-renderer` keeps renderer `require('fs')` calls (Config.js, SaveManager.js, DLCManager.js, game.js) working post-bundle.
- **ESM leaf-file migration** (#2 ‚Üí #1/#3 unlock) ‚Äî `Utils.js`, `Managers/SpatialHash.js`, `Constants.js`, and `Managers/SaveManager.js` are now native ES modules with explicit `export` lists. Each retains its `window.X = X` shim so classic-script callers see no behavior change. `game.html` marks these four files as `<script type="module">`; every other script tag now carries `defer` so module + classic execute in document order after parsing. Inline `<script>` blocks that read `APP_VERSION` are wrapped in `DOMContentLoaded` listeners to honour the new ordering. This is the first phase of the longer ESM migration (Managers, Entities, UI, game.js, DLCs in future sessions).
- **ESLint + Prettier + custom lint rule** (#13, #14) ‚Äî flat ESLint 9 config in `eslint.config.mjs` with warnings-only policy (zero errors, lint never blocks). New local plugin `eslint-plugin-5freunde/` with one rule `no-foreach-splice` that detects the `arr.forEach((e, i) => arr.splice(i, 1))` index-skip bug class ‚Äî the audit pass on 2026-05-10 found 11 of these; this run found 8 more. `.prettierrc` documents project style (4-space, single quotes, 120-col). `npm run lint` / `npm run lint:fix` scripts.
- **Vitest unit-test harness** (#12) ‚Äî `tests/` directory with 38 tests across 6 files: `saveManager.test.js` (schema migration invariants), `mulberry32.test.js` (determinism + uniformity), `spatialHash.test.js` (broad-phase query correctness), `hermiteInterp.test.js` (snapshot interpolation edge cases), `plausibility.test.js` (leaderboard anti-cheat caps), `rateLimit.test.js` (token-bucket refill semantics). New `server/anticheat.js` extracts the plausibility + rate-limit helpers from `server.js` so they're unit-testable without booting the WS server. `npm test` runs Vitest then the existing `server/simulation/parityTest.js` (80/80 assertions passing after the AirHero stub fix below).
- **GitHub Actions CI** ‚Äî `.github/workflows/test.yml` renamed to a `CI` workflow that runs on every push/PR: dependency install, the existing legacy-globals lint, ESLint, Vite production build, and the full test suite (Vitest + parity).

### Fixed
- **parityTest level-up flow typo** ‚Äî `server/simulation/GameSession.js:211` used `this.world.player` / `this.world.player2` instead of `this._world.player(2)`. The level-up choice handler crashed every test run since the World refactor. Caught by the Vitest harness pass.
- **AirHero `Cannot convert object to primitive value` on server tick** ‚Äî `dlc/waker_of_winds/AirHero.js:777` reads `ctx.canvas.width` for HUD layout. The server-side `_noopCtx` Proxy in `server/simulation/loader.js` returned itself for every property access including `Symbol.toPrimitive`, so arithmetic operations on stub fetches (`screenW / 2`) crashed. Added explicit `Symbol.toPrimitive` / `valueOf` / `toString` handlers on the Proxy that yield `0` / `''`. AirHero tick now runs cleanly server-side.
- **Removed accidental devDep** ‚Äî `install@^0.13.0` was sitting in `devDependencies` from an earlier mistype (per improvements #153). Dropped during the npm install pass.

### Changed
- **`server/server.js` anti-cheat helpers** ‚Äî `plausibilityReject` and the token-bucket logic now live in `server/anticheat.js`. `server.js` imports them as a thin wrapper that adds the Express middleware shape. No behavior change.

- **End-of-run breakdown** (#160) ‚Äî victory + game-over screens now render a third section below the existing Run Performance / Final Build columns:
  - **Damage by Source** ‚Äî proportional bars showing projectile / melee / special / DoT contributions. Each damage site (`bumpDamageSource`) buckets into its source so future weapon types add a row without code changes.
  - **Upgrades Picked** ‚Äî full timeline `W{wave} {title} {time}` of every level-up choice the local player made. Hero-specific upgrade titles are looked up via `HERO_LOGIC[type].upgradePool` when present.
  - **Key Moments** ‚Äî boss kills + level-ups timestamped per wave. Future hooks (near-deaths, special-form activations) can push into `currentRunStats.keyMoments`.
- **Reduced Motion accessibility mode** (#134) ‚Äî new Options toggle (`gameConfig.reducedMotion`) that disables camera shake, weather particle overlays, and the pulsing red low-HP vignette. Static vignette + damage numbers remain so combat feedback stays intact. Helper `isReducedMotion()` is exposed for future renderers to gate on.
- **Colorblind correction modes** (#50) ‚Äî new Options control cycling through Deuteranopia / Protanopia / Tritanopia daltonization filters, applied via SVG `<feColorMatrix>` on the canvas element. Off by default. The filter definitions live in `game.html` (`#cb-*`) so they apply without any draw-call changes.
- **Crash reporting** (#97) ‚Äî new `Managers/CrashReporter.js`. Captures `window.onerror`, `unhandledrejection`, and a 30-entry breadcrumb ring of `console.warn`/`error`. Posts coalesced reports (1 per stack per minute) to `POST /api/crash`. Opt-out via Options ‚Üí "Crash Reports". Server endpoint appends JSONL to `data/crashes.jsonl`; admin dashboard sees last 200 entries at `GET /api/admin/crashes`. No PII ‚Äî username stripped, IP pseudonymised before storage.
- **Daily seed leaderboard tagging** (#61) ‚Äî daily/weekly runs now stamp their `dailySeed` (YYYYMMDD / YYYYWW) on leaderboard POSTs. New server column `scores.daily_seed`; `GET /api/leaderboard?dailySeed=20260510` returns per-seed leaderboards. Mutator RNG was replaced from a Math.sin-based seed (biased) to `mulberry32` (uniform). New `getSeededRng()` / `initSeededRng()` globals give future arena/drop code an opt-in seeded RNG bound to today's seed so all players on the same day share identical rolls.
- **Save schema versioning + migrations** (#10) ‚Äî `SaveManager.SCHEMA_VERSION` + `MIGRATIONS` table. Each load reads `data.version`, runs every applicable migration in sequence, and writes a one-shot pre-migration backup (`save_data.json.pre_v{X}_to_v{Y}.bak` on disk, separate localStorage key on web) before the first write. Saves without a version field are treated as v0. `defaultSaveData.version` is now stamped on every write.
- **Online anti-cheat session tokens** (#87) ‚Äî server issues a per-user signed session token at `START_ONLINE_GAME`. The client returns it on `POST /api/leaderboard`; the server clamps the client-claimed score downward to its own authoritative `wave`/`score` and marks the entry `verified=1`. Unverified submissions go through a heuristic plausibility check (wave/score/time ratios) and are tagged `verified=0`. New scores column `verified`. `GET /api/leaderboard?verified=1` filters to server-confirmed entries.
- **Rate limiting on auth + leaderboard + crash endpoints** (#88) ‚Äî new lightweight token-bucket middleware (no external dependency). Per-IP limits: `/api/register` 5/hour, `/api/login` 10/minute, `/api/leaderboard` POST 10/hour, `/api/crash` 20/10min. Buckets are swept every 5 minutes to bound memory.
- **HTTPS / WSS support** (#90) ‚Äî when `TLS_CERT_PATH` + `TLS_KEY_PATH` env vars are set, the server boots `https.createServer` with the supplied cert (and optional `TLS_CA_PATH` chain) and the WebSocket upgrades to WSS automatically. Falls back to plain HTTP for local dev. The client's `serverUrl` regex (`http‚Üíws`) already handles the `https‚Üíwss` transition.
- **WebSocket permessage-deflate compression** (#158) ‚Äî enabled on the server's `ws.Server` with conservative zlib settings (level 3, 256-byte threshold so PING/INPUT skip compression). Cuts snapshot bandwidth ~50‚Äì70% on internet links.
- **Cubic-Hermite snapshot interpolation** (#155) ‚Äî client-side `_onlineInterpBuf` was a linear lerp; now a Catmull-Rom cubic Hermite with tangents derived from neighboring snapshots. Smoother during dash transitions where linear interp produced a visible corner at each snapshot boundary. Boundary samples (first/last in buffer) still fall back to linear so the edges stay well-defined.
- **Variable server tick rate** (#156) ‚Äî GameSession nominal tick stays at 30 Hz (33 ms) but drops to 20 Hz (50 ms) when `enemies + projectiles > 180`, with 140-entity hysteresis on the way back up. `TICK_FRAMES` is now per-tick instance state so simulated game speed stays constant across tick-rate changes (sub-step count scales). The tick loop is now self-rescheduling (`setTimeout` chain) rather than `setInterval`.
- **MTU-aware snapshot chunking** (#157) ‚Äî snapshots with more than 100 entities are split across 2‚Äì3 `SNAPSHOT` messages tagged with `chunk: {seq, idx, of}`. Client `_onlineHandleSnapshot` buffers chunks by `seq`, reassembles when all parts arrive, and evicts stale partials after 500 ms. Small snapshots ship unchanged (backward-compatible ‚Äî the chunk field is optional).
- **Live authoritative score relay** ‚Äî GameSession now calls an optional `onTickStats` callback every tick, letting server.js keep `_sessionScores` fresh so leaderboard verification works even if the client posts before the WS `GAME_OVER` message lands.
- **Spatial hash infrastructure** (#19) ‚Äî new `Managers/SpatialHash.js` (uniform-grid broad phase, 128 px cells). Rebuilt once per frame from the enemy list and queried by AOE radius scans (explosive projectile splash, wall-hit explosions, swarm self-destruct). Full nested projectile-vs-enemy loop inversion is left for a follow-up pass.
- **DLC parallel load** (#29) ‚Äî `DLCManager.init` now awaits enabled DLC loads via `Promise.all` instead of sequential awaits. Each DLC ships ~50 KB of scripts; on 5+ enabled DLCs this cuts startup wait by hundreds of milliseconds on internet-hosted assets.
